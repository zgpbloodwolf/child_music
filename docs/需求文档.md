## 1. 项目概述

- **定位**:手机端**儿童启蒙音频**应用,内容涵盖**儿歌、古诗、三字经、故事**四类,核心是「听 + 播放控制 + 后台播放」。
- **目标平台**:App(iOS/Android)优先,可同时编译到 H5 与微信小程序(见 [CLAUDE.md](../CLAUDE.md) 第 1、6 节)。
- **MVP 聚焦**:**内置曲库离线播放** —— 用户安装即可听,无需联网。
- **开发顺序**:内置曲库版本先跑通;联网接入第三方音频为后续阶段。

---

## 2. 核心决策

| 维度 | 决策 | 说明 |
| --- | --- | --- |
| 内容来源 | **先内置音频**(MVP) | 内置儿歌/古诗/三字经/故事,安装即可听;联网接入为后续阶段 |
| 内容分类 | **四类**:儿歌 / 古诗 / 三字经 / 故事 | 预设歌单按此分类,首页可切换浏览 |
| 后端服务 | **暂无后端,纯前端** | 收藏、歌单等用 `uni.setStorageSync` 本地存储;后续按需再加 |
| 调试平台 | **先 H5 开发** | H5 端起步最快、调试最方便,稳定后再适配 App / 微信小程序 |
| 内置资源 | **用户自备** | 音频文件(.mp3 等)+ 元数据(名称/作者/封面/歌词)由用户准备 |

## 3. MVP 功能范围

### 3.1 核心功能(默认包含)

**播放器控制**
- 播放 / 暂停
- 上一首 / 下一首
- 播放模式:顺序播放、单曲循环、随机播放
- 进度条拖动跳转(`bgAudioManager.seek`,UI 进度条节流 ≥250ms)
- **后台播放 + 锁屏控制 + 系统通知栏**(统一走 `uni.getBackgroundAudioManager()`)
- 所有播放逻辑收敛到 `src/store/player.ts`(单一数据源)
- iOS 后台播放需在 `manifest.json` 配置 `UIBackgroundModes: ['audio']`

**内置曲库列表**
- 全部内置音频列表,展示封面、名称、作者/来源
- 点击即播放,并加入当前播放队列

**播放页**
- 封面(大图)、名称、作者/来源
- 进度条与当前/总时长
- 播放控制按钮组

### 3.2 增强功能(均已纳入 MVP)

- **预设分类歌单**:内置内容按类型分为「儿歌」「古诗」「三字经」「故事」,首页可切换浏览,点击进入歌单详情。
- **歌词/文本同步显示**:儿歌、古诗、三字经随播放进度滚动、高亮当前句;故事类无逐句文本,播放页显示「暂无歌词」。
- **本地收藏(「我喜欢的」)**:点红心收藏,单独列表,数据存本地存储。
- **本地搜索**:在内置曲库内按名称 / 作者 / 专辑快速搜索。

## 5. 技术约束(摘要,详见 CLAUDE.md)

- 框架:uni-app + Vue 3 `<script setup>` 组合式 API + Vite
- 语言:TypeScript(禁止新增裸 `.js`)
- 状态管理:Pinia(模块化,如 `player` / `library`)
- 样式:SCSS + `rpx`(750 设计稿基准),颜色用 `uni.scss` 变量
- 包管理:pnpm
- 跨端:平台差异用条件编译;禁止通用代码引用 `document` / `window`;本地存储用 `uni.setStorageSync`
- 音频:**强制**使用 `uni.getBackgroundAudioManager()`,禁止用 `innerAudioContext` 做主播放(见 [CLAUDE.md](../CLAUDE.md) 第 4 节)
- 语言:界面文案、注释、文档一律全中文

## 7. 内置曲库数据结构(已实现,供填充资源参考)

### 7.1 音频对象结构

定义在 `src/types/song.ts`,数据源在 `src/data/songs.ts`:

```ts
interface Song {
  id: string          // 唯一标识
  name: string        // 名称(儿歌名/古诗名/故事名等)
  artist: string      // 作者/来源(如「李白」「经典儿歌」「格林童话」)
  album?: string      // 专辑/合集(可选,如「唐诗」)
  cover: string       // 封面图路径,如 /static/covers/gs001.jpg(为空时组件色块占位)
  src: string         // 音频文件路径,如 /static/audio/gs001.mp3
  duration?: number   // 时长,单位秒(可选,由播放器回调填充)
  lyric?: string      // 歌词/文本(lrc 字符串,内嵌;故事类可留空)
  category?: string[] // 所属分类 id,如 ['poetry'](对应儿歌/古诗/三字经/故事)
}
```

> 实现说明:歌词采用**内嵌字符串**(`lyric` 字段),而非独立 lrc 文件,跨端最稳。

### 7.2 资源目录

```
src/static/
├── audio/         # 音频文件(.mp3 等),对应 Song.src
└── covers/        # 封面图(.jpg / .png),对应 Song.cover

src/data/
├── songs.ts       # 内置音频索引(参与编译,数据源)
└── playlists.ts   # 预设分类歌单
```

> 说明:数据索引导出文件放在 `src/data/`(参与编译),不放 `static/`(`static/` 不参与编译,见 [CLAUDE.md](../CLAUDE.md) 第 3 节)。音频/封面等纯资源放 `src/static/`。

### 7.3 预设分类歌单

定义在 `src/data/playlists.ts`,四类:

```ts
interface Playlist {
  id: string         // 分类 id:children / poetry / classics / story
  name: string       // 名称:儿歌 / 古诗 / 三字经 / 故事
  description?: string
  songIds: string[]  // 包含的音频 id 列表
}
```
